Spring Boot & REST:

RESTful Web Service & Spring

Quick overview on REST: What does it mean to have a RESTful web service, and how does Spring Boot help us in making our application RESTful?

    (In a nut shell)
  • REST is a convention that allows for a bit of an enhancement when it comes to the client-server relationship. RESTful implementations use the HTTP protocol verbs like GET, PUT, POST, and DELETE.
  • When the client sends a request our way, you can be "REST" assured that we can take care of it on our server side. Spring helps us build RESTful services with its useful annotations that take care of incoming requests from the client. Whether it is a GET, PUT, POST, or DELETE, these requests are handled in the controller class (annotated with @Controller or @RestController depending on the type of data you want returned). Simple Spring annotations can help direct request traffic to its appropriate destination.
  • REST is just one area where Spring/Spring-boot can make our lives as developers easier. There are several benefits to using this framework in our projects, like:
    • REST is just one area where Spring/Spring Boot can make our lives as developers easier. There are several benefits to using this framework in our projects, for example: Spring achieves loose coupling through dependency injection (as seen in my "Swap Me" project)
    • Spring reduces a lot of boilerplate code especially when it comes to ORM (Hibernate), JPA, data marshalling, validation, and creating objects via form data (again, much of which is used in my "Swap Me" web application)
    • Spring removes the need of having an instance of a web server (like Apache Tomcat) installed on your machine
    • Spring provides support for MVC-based applications
    • Spring provides enterprise-level security with Spring Security
    • And much, much, more

MySQL:

Relational database management system (RDBMS)

Quick overview on RDBMS: A relational database is a type of database that stores data in tables, these tables can have relationships with other tables that can contain data that relate to one another. These tables are divided into rows and columns, and if we want to retrieve the data within these tables, we can use the Structured Query Language (SQL) to do just that. MySQL is one of these relational database management systems, which is a very popular one and happens to also be open source.

  • Here are few reasons why mySQL is so great:
    • Data security, data security is so good with mySQL that even tech giants like Amazon and Twitter rely on it
    • Scalability, mySQL automatic sharding (partitioning) to scale horizontally, & customization for unique database server requirements
    • Extremely easy to use and set up (about 30 min or less)
    • High performance, mySQL provides "Full-text indexing" (able to find data in a table without having to scan rows & columns)
    • And much more
